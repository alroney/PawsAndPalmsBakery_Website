Object Oriented Programming (OOP)

Definition: OOP is a programming paradigm that revolves around the concept of objects which are instances of classes. It organizes code into objects each containing both data (attributes) and behavior (methods).

Key Principles: 4 Pillars of OOP

  • Encapsulation

    Encapsulation is the bundling of attributes and methods or functions into a single unit or object.

    • Getters/Setters: also known as accessor and mutator methods, Getters are methods used to retrieve the values of private attributes, while setters are methods used to modify or set the values of private attributes. By using getters and setters, we can control how the data is accessed and modified, enforcing constaints or validation rules if necessary.

    • Access Control and Access Modifiers: are mechanisms used in to specify the accessibility of classes, methods, and attributes. Common access modifiers include:
      1. Public

        Public members are accessible from any part of the program, both within the class and outside of it. Best time to Use

        1. I would use public encapsulation when I want attributes and methods to be freely accessible from anywhere in the program, providing a clear interface for external use. For example, in a utility class that provides commonly used functions or constants.

      2. Private

        Private members are accessible only within the class where they are declared. They cannot be accessed or modified from outside the class. Best time to Use

        1. I would use private encapsulation when I want to hide implementation details and control access to sensitive data within a class. For instance, in a banking application, I would use private encapsulation to protect attributes like account balances and passwords.

      3. Protected

        Protected members are accessible within the class where they are declared and in subclasses, but not outside of the class hierarchy. Best time to Use

        1. I would use protected encapsulation when I want to share attributes or methods among related classes within a class hierarchy while still restricting access from outside the hierarchy. For instance, in a software framework where I have a base class with certain attributes or methods that should be accessible to subclasses but not to external classes. This allows for code reuse and specialization within the hierarchy while maintaining encapsulation and ensuring that details are hidden from unrelated classes.

    • Access Modifiers: help in enforcing encapsulation by controlling the visibility and accessibility of class members. They allow deelopers to specify which parts of a class should be exposed to the external world and which should remain hidden.

    • Class Binding: also known as data binding, refers to the process of associating data with its behavior. In OOP, a class binds attributes and methods together, forming objects. This binding ensures that the attribute and method are encapsulated within the same unit, promoting modularity and code organization.

  • Inheritance

    Inheritance refers to the mechanism where a class (aka subclass) can inherit attributes and methods from another class (aka superclass).

    • Hierarchical Inheritance: occurs when multiple subclasses inherit from the same superclass. This forms a hierarchy, where each subclass inherits the attributes and methods of the superclass while also having the ability to define its own additional attributes and methods.

    • Parent VS Child Classes:
      1. Parent Class:
        1. - The parent class is the class from which other classes inherit attributes and methods.
        2. - It serves as a template for creating subclasses.
        3. - It defines common behavior and characteristics shared by its sbuclasses.
      2. Child Class:
        1. - The child class is a class that inherits attributes and methods from its parent class.
        2. - It can also dfines its own additional attributes and methods.
        3. - It extends or modifies the behavior inherited from the parent class to create specialized functionality

    • Summary: In summary, inheritance allows for code reuse, promotes modularity, and enables the creation of specialized classes with uniques behaviors by establising relationships between classes based on their similarities and differences.

  • Abstraction

    Abstraction is a principle simplifying complex systems by modeling classes based on real-world entities and focusing on essential attributes and behaviors while hiding unnecessary details.

    • Interface: is a collection of abstract methods that define a contract for classes that implement them. Interfaces specify what methods a class must implement, but they do not provide the implementation details. Classes that implement an interface must provide concrete implementations for all the methods declared in the interface.

    • Abstract Classes: are classes that cannot be instantiated on its own and may contain one or more abstract methods. Abstract classes serve as templates for creating conrete subclasses. They can contain both abstract methods (methods without a body) and concrete methods (methods with a body). Abstract classes can also have attributes and constructors.

    • Abstract Methods: are methods declared within an abstract class or an interface that does not have an implmentation. Abstract methods define a method signature without providing the method body. Subclasses or classes implementing the interface must provide a concrete impplementation for all abstract methods declared in the superclass or interface.

    • Constructors: are special methods used for initilizing objects of a class. Constructors are called automatically when an object is created. They typically initialize the object's attributes or perform any necessary stup operations. Constructors can be parameterized (accepting arguments) or default (no arguments).

    • Default Constructors: are constructors with no parameters. It is automatically provided by the compiler if no explicit constructos are defined in the class. Default constructors initialize the object's attributes to their default values or perform any necessary setup operations. They are called automatically when an object is created without providing any arguments.

    • Summary: Abstraction allows developers to create generalized models by focusing on essential attributes and behaviors, while hiding unnecessary details. Interfaces define a contract for classes to implements. Abstract classes serve as templates for create concrete subclasses. Abstract methods define method signatures without implementations. Constructors initilize objects, and default constructors are provided by the compiler if no explicit constructors are defined.

  • Polymorphism

    Polymorphism is the concept which allows object to be treated as instances of their superclass or as instances of their subclasses. This means a single interface can represent multiple forms.

    • Method Overriding

      occurs when a subclass provides a specific implementation of a method that is already defined in its superclass. The subclass method has the same signature (name and parameters) as the superclass method, but provides a different implementation. Method overriding allows for runtime polymorphism.


    • Method Overloading

      occurs when multiple methods in the same class have the same name but different parameter lists. Each overloaded method provides a different behavior based on the parameters it accepts. Method overloading allows for compile-time polymorphism.


    • Runtime VS Comile-time:
      • Runtime: is achieved through method overriding. This allows the specific method to be executed to be determined dynamically at runtime based on the actual type of the object.
      • Compile-time: is achieved through mehtod overloading. This allows the appropriate method to be called to be determined statically at compile time based on the method's signature.

    • Covariance VS Contra-Variance:
      • Covariance: refers to the principle in type systems where a function or method is allowed to return a more derived (specific) type than originally specified, or where a type can be replaced by its subtype. This concept enables the substitution of more specific types in place of less derived types, particularly in return types of functions and output types of objects.
      • Contra-Variance: refers to the principle in type systems where a function can accept arguments of a less derived (more general) type than originally specified. It allows for the substitution of general types in place of more specific types in certain contexts, particularly in function parameteres.

    • Classes, Properties, and Parameters:
      • Classes: are blueprints for creating object in OOP. They define attributes and methods that characterize object of the class.
      • Properties: are attributes of a class that define the state of objects. They encapsulate data and provide a way to access and modify the state of objects.
      • Parameters: are variables that are passed to methods or constructors. They provide a way to pass data to methods or constructors for processing.

    • Summary

      Polymorphism allows for flexibility and reuse in OOP through method overriding and method overloading. Runtime polymorphism is achieved through method overriding, while compile-time polymorphism is achieved through method overloading. Covariance and contra-variance allow for substitution of types in certain contexts. Classes define the structure and behavior of objects, properties encapsulate data, and parameters provide a way to pass data to methods or constructors.


Structured Query Language (SQL) Syntax

Definition: Structured Query Language (SQL) is used for managing and manipulating relational databases. It provides a set of commands for performing various operations such as querying data, updating data, creating and modifying database schematics, and managing database transactions.

Key Principles:

  • Basic Queries:

    Basic SQL queries are used to retrieve data from a database.

    • SELECT (FROM):
    • WHERE:
    • BETWEEN:
    • LIKE:
    • ORDER:
    • JOIN:

  • Data Manipulation Language (DML):

    Data Manipulation Language statements are SQL commands used to manipulate data in a database.

    • INSERT: to add new rows.
    • UPDATE: to modify existing rows.
    • DELETE: to remove rows.
    • MERGE: to perform a comination of INSERT, UPDATE, and DELETE operations based on a specified condition.

  • Primary/Foreign Keys:
    • Primary Key: is a column or combination of columns that uniquely identifies each row in a table. It ensures that each row in a table is unique and serves as the primary means of identifying records.
    • Foreign Key: is a column or combination of columns that establishes a link between data in two tables. It enforces referential integrity by ensuring that values in the foreign key column(s) match values in the primary key column(s) of another table.

  • Normalization:

    Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves breaking down large tables into smaller, related tables and dfining relationships between them. Normalization helps eliminate data anomalies such as insertion, update and deletion anomolies, and ensures that data is stored efficiently.


  • Data Definition Langauge (DDL):

    Data Definition Language statements are SQL commands used to define and manage the structure of database objects such as tables, indexes, views, and constraints.

    • CREATE TABLE: to create objects.
    • ALTER TABLE: to modify objects.
    • DROP TABLE: to delete objects.

  • Joins:

    Joins are SQL operations used to combine data from multiple tables based on related columns between them.

    • INNER JOIN: returns rows when there is a match in both tables.
    • LEFT JOIN: returns all rows from the left tables and matching rows from the right tables.
    • RIGHT JOIN: returns all rows from the right table and matching rows from the left table.
    • FULL JOIN: returns rows when there is a match in either table.

  • SQL Summary:
    • SQL is used for managing and manipulating relational databases. It provides syntax for querying data, manipulating data, defining database structure, and manging transactions. All of which is done through basic queries, data manipulatinon language statements, primary and foreign keys, normalization, data definition language statements, and joins.

Data Structures

Definition: Data Structures are specialized formats for organizing, storing, and manipulating data efficiently. They provide different ways of organizing data to suit various applications and requirements.

Key Principles:

  • Lists

    A lists is a linear data structure that stores elements sequentially. Lists allow for dynamic resizing and typically support operations such as insertion, deletion and traversal. Common type of lists include:

    • - Arrays
    • - Linked Lists
    • - Dynamic Arrays
    • - Vectors

  • Queues:

    A queue is a linear data structure that follows the First-In-First-Out (FIFO) principle, where elements are added to the rear and removed from the front. Queues supports operations such as enqueue (addition) and dequeue (removal). They are commonly used in scenarios like task scheduling, job processing, and breadth-first search algorithms.

    • - Simple Queue
    • - Circular Queue
    • - Priority Queue
    • - Double Ended Queue

  • Maps:

    A map (aka dictionary or associative array) is a collection of key-value pairs where each key is associated with a value. Maps allow for efficient lookup and retrieval of values based on keys.

    • - HashMap
    • - TreeMap
    • - LinkedHashMap

  • Sets:

    A set is a collection of unique elemetns with no duplicates. Sets support operations such as addition, removal, and intersection. They are useful for tasks like removing duplicates from a list, testing membership, and performing set operations like union and intersection.

    • - HashSet
    • - TreeSet
    • - LinkedHashSet

  • Equals Method:

    The equals() method is a method that is used to compare the equality of objects. It is typically overriden in user-defined classes to provide custom comparison logic based on the object's attributes. The equals() method is commonly used to compare objects for equality in scenarios like searching, sorting, and hashing.


  • HashCode:

    The hashCode() method is a method that is used to generate a unique integer value (hash code) for an object. Hash codes are used in data structures like hash tables and hash sets to efficiently store and retrieve objects. The hashCode() method is often overridden in conjunction with the equals() method to ensure consistent behavior when objects are used as keys in hash-based collections.


  • Summary:

    Data Structures are essential tools for organizing and manipulating data efficiently. Lists, queues, maps, and sets are common types of data structures used in various applications.


Define

  • if

    The "if" statement is a control flow statement used to execute a block of code only if a specified condition is true.


  • else

    The "else" statement is used in conjunction with the "if" statement to specify an alternative block of code to execute if the condition specified in the "if" is false.


  • For Loop

    A "for" loop is a control flow statement used to repeatedly execute a block of code a specified number of time. It typically consists of an initialization, a condition, and an increment or decrement operation.


  • While Loop

    A "while" loop is a control flow statement used to repeatedly execute a block of code as long as a specified condition is true.


  • Mathematical Operations

    Mathematical operations refer to arithmetic operations performed on numeric data types. Common mathematical operations include addition (+), subtraction (-), multiplication (*), division (/), modulus (%) and exponentitation (^ or **).


  • Comparison Operations

    Comparison operations are used to compare two values and determine their relationship. Common comparison operations include equality (==), inequality (!=), greater than (>), lessthan (<), greater than or equal to (>=) and less than or equal to (<=).


  • Exception Handling

    Exception handling is a programming construct used to handle runtime errors or exceptional conditions that may occur during program execution. It involves catching, handling, and possibly recovering from errors to prevent program termination or data corruption.


  • Increment Operators

    Increment operators are used to increment the value of a variable by a specified amount. Common increment operators include ++ (increment by 1) and += (increment by a specified value).


  • Big-O Notation

    Big-O Notation is a mathematical notation used to describe asymptotic behavior (performance or complexity) of an algorithm as the input size approaches infinity. It provides an upper bound on the worst-case time or space complexity of an algorithm.


  • Performance vs Scalability
    • - Performance refers to the speed or efficiency with which a system or program executes a task under specific conditions. It focuses on optimizing resource usage to achieve faster execution times.

    • - Scalability refers to the ability of a system or program to handle increasing amounts of work or users without sacrificing performance. It focuses on designing systems that can grow and adapt to handle larger workloads or user demands.

  • Agile vs Waterfall Methodologies
    • - Agile methodology is an iterative and incremental approach to software development, which enhances the continuous view of changing requirements to give flexibility and open ways for customer involvement and interaction. It represents the best priority of delivering working software frequently and adapting to the changing requirements constantly.

    • - Waterfall methodology is a tradition linear-sequential approach to software development, composed of distinct phases including requirements analysis, design, implementation, testing and maintenance among others. It is a plan-driven, rigid process where once the development is started, flexibility in accommodating changes is minimum.
